Unity Journal 2 - C#, .NET, IL2CPP

2018/01/22

Categories: Journal Unity Journal Tags: Unity C#

What is Managed and Unmanaged Code?

  • Managed code runs in the Common Language Runtime (CLR) and compiles to an intermediate language (IL)
  • Unmanaged is outside the CLR, and compile directly to native machine code

Unity Scripting Backend

  • Replacing Mono with in house .NET runtime IL2CPP
  • Generates C++ source code from assemblies, then relies on existing C++ compilers to create native binaries
  • Virtual Machine (VM) provides extra such as Garbage Collection (GC), metadata, and platform specific resources
  • Leverages existing compilers and removes effort in porting Mono to every platform
  • IL2CPP will fast forward Unity to modern .NET features

.NET Standard 2.0

  • .NET Core is a platform which implements .NET Standards
  • Support for an increased number of APIs

Unity Platform Support

  • C++ Engine, compiled natively for each platform
  • Extensive use of third party libraries (for physics, audio, lighting etc)
  • Leverages scripting backend (Mono or IL2CPP) to run C# scripts in native environment

IL2CPP Internals

  • Two parts: ahead-of-time (AOT) compiler, runtime library to support VM
  • Runtime is designed to be independent of a specific garbage collector, meaning Unity could integrate a different one than what currently ships
  • Still relies on C# standard library code
